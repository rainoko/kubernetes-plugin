This setting controls how slave pods are retained after the Jenkins build completes for this pod template. Values other than "Default" will override the plugin's Pod Retention setting. The following retention policies are provided:

  1. Always - always keep the slave pod.
  2. Default - use the Pod Retention setting for the plugin.
  3. Never - always delete the slave pod.
  4. On Failure - keep the slave pod if it fails during the build.

Note: Kubernetes administrators are responsible for managing any kept slave pod. These will not be deleted by the Jenkins Kubernetes plugin.
